# 重构 读书笔记 3 封装

  分解模块最重要的标准，也许就是识别出那些模块应该对外界隐藏的小秘密。

  封装记录；曾用名，以数据类取代记录。

organization = { name: 'Acme Gooseberries', country: 'GB' }

class Organization {
  constructor(data) {
    this._name = data.name
    this._country = data.country
  }

  get name() {
    return this._name
  }

  set name(arg) {
    this._name = arg
  }

  get country() {
    return this._country
  }

  set country(arg) {
    this._country = arg
  }
}

  程序中间常常需要互相传递嵌套的列表或散列映射结构，这些数据结构后续经常需要被序列化 json 或 xml。这样的嵌套结构同样值得封装，这样，如果后续其结构需要变更或者需要修改记录内的值，封装能够帮我更好地应对变化。

class Person {
  get courses() {
    return this._courses
  }

  set courses(aList) {
    this._courses = aList
  }
}

  提炼类：内联类。

class Person {
  get officeAreaCode() {
    return this._officeAreaCode
  }

  get officeNumber() {
    return this._officeNumber
  }
}

// 优化
class Person {
  get officeAreaCode() {
    return this._telephoneNumber.areaCode
  }

  get officeNumber() {
    return this._telephoneNumber.number
  }
}

class TelephoneNumber {
  get areaCode() {
    return this._areaCode
  }
  get number() {
    return this._number
  }
}

  替换算法。

function foundPerson(people) {
  for (let i = 0; i < people.length; i++) {
    if (people[i] === 'Don') {
      return 'Don'
    }
    if (people[i] === 'John') {
      return 'John'
    }
    if (people[i] === 'Kent') {
      return 'Kent'
    }
  }
  return ''
}

// 优化
function foundPerson(people) {
  const candidates = ['Don', 'John', 'Kent']
  return people.find((p) => candidates.includes(p) || '')
}
