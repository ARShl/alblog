# javascript 忍者秘籍 1

  函数是第一类对象，或者说它们被称作一等公民。函数式方式让我们能把函数座位一个单独实体来创建，正像我们对待其他类型一样，创建它，作为参数传入一个方法并将它作为一个参数来接收。函数就这样显示了它一等公民的地位。

  记忆化是一种构建函数的处理过程，能够记住上次计算结果。如果另外一个调用也使用相同的参数，我们则可以直接返回上次存储的结果而不是再计算一遍。像这样避免既重复又复杂的计算可以显著的提高性能。对于动画中的计算，搜索不经常变化的数据或任何耗时的数学计算来说，记忆化这种方式是十分有用的。

function isPrime(value) {
  if (!isPrime.answers) {
    isPrime.answers = {}
  } // 创建缓存
  if (isPrime.answers[value] !== undefined) {
    return isPrime.answers[value]
  } // 检查缓存值

  var prime = value !== 0 && value !== 1
  for (var i = 2; i < value; i++) {
    if (value % 2 == 0) {
      prime = false
      break
    }
  }

  return (isPrime.answers[value] = prime)
}

  这个方法有两个优点：由于函数调用时会寻找之前调用所得到的值，所以用户最终会乐于看到所获得的性能收益。它几乎是无缝发生在后台，最终用户喝页面作者都不需要执行任何特殊请求，也不需要做任何额外初始化，就能顺利进行工作。当然这种方法并不是像玫瑰喝提琴一样完美，还是要权衡利弊。任何类型的缓存都必然会为性能牺牲内存。纯粹主义者会认为缓存逻辑不应该喝业务逻辑混合，函数或方法只需要把一件事做好。对于这类问题很难做负载测试或者估计算法复杂度，因为结果一拉利鱼函数之前的输入。

  在讨论函数时，经常使用的术语实参（argument）和形参(parameter)几乎可以互换，就好像它们差不多时同一种事物。形参是我们定义函数时所列举的变量。实参时我们调用函数时所传递给函数的值。函数形参时在函数定义时指定的，而且所有类型的函数都能有形参。从另一方面说，实参则与函数的调用相联系。它们时函数调用时所传递给函数的值。当函数调用时提供了一系列实参，这些实参酒会以形参在函数中定义的顺序被赋值到形参上。第一个实参赋值到第一个形参，第二个实参赋值到第二个形参上，以此类推。

  实参的数量大于形参的数量并不会抛出错误。如果实参的数量大于形参，那么额外的实参不会赋值给任何形参。如果形参的数量大于实参，那么哪些没有对应实参的形参会被设为 undefined。

  在 es6 中可以使用剩余参数(reset parameters)。为函数的最后一个命名参数前加上省略号(...)前缀。这个参数酒变成了一个叫作剩余参数的数组。数组内包含着传入的剩余的参数。只有函数的最后一个参数才能是剩余参数。试图把省略号放在不是最后一个形参的任意形参之前都会报错，错误以 SynataxError:parameter after rest parameter 的形式展现。

  arguments 参数是传递给函数的所有参数集合。无论是否有明确定义对应的形参，通过它我们都可以访问到函数的所有参数。借此可以实现原生 js 并不支持的函数重载特性，而且可以实现接收参数数量可变的可变函数。arguments 对象有一个 length 属性，表示实参的确切个数。通过数组索引的方式可以获取单个参数的值。

  js 单线程执行模型一种不可避免的结果是，同一时刻只能执行一个任务。这意味着所有任务都必须在队列中排队等待执行时机。
